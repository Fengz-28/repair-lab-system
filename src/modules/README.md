# Modules

Los modulos contienen dominio y casos de uso. No deben depender de proveedores concretos como SMTP, WhatsApp, Google Calendar, Ollama o n8n.

Modulos previstos:

- `customers`
- `devices`
- `intake`
- `tickets`
- `appointments`
- `catalog`
- `inventory`
- `billing`
- `communications`
- `crm`

Cada modulo debe documentar entidades, permisos, validaciones, eventos emitidos, auditoria y dependencias internas.
